# USER.md Example

## Purpose

This file defines the human the agent is helping.

It gives the agent context about:
- who the person is
- what they do
- what they care about
- how they like to work
- what constraints or goals matter most

A strong `USER.md` should help the agent make better decisions on the user’s behalf.

---

# USER.md

- **Name:** Alex Rivera
- **What to call them:** Alex
- **Timezone:** America/Toronto
- **Location:** Toronto, Canada
- **Primary role:** Founder and operator

## What Alex does

Alex runs a small service business that combines consulting, automation, and content.
Their work includes:
- client delivery
- sales follow-up
- content creation
- operations
- community building

## Core goals

Alex wants help with:
- reducing admin work
- staying organized
- following up faster
- creating content more consistently
- improving revenue visibility
- building systems that scale

## Current priorities

Right now, the highest priorities are:
1. closing new client work
2. keeping delivery organized
3. publishing useful content weekly
4. building repeatable internal systems

## Communication preferences

### Tone preferences
- direct
- practical
- clear
- low fluff

### Response preferences
- concise by default
- bullets are better than long essays
- explain tradeoffs when needed
- give a recommendation, not just options

### Things to avoid
- generic business advice
- vague motivation language
- long theory without action
- overexplaining obvious basics

## Working style

Alex tends to move quickly and values momentum.
That means the agent should:
- help reduce friction
- keep things organized
- prefer workable drafts over perfection
- make reasonable assumptions when safe
- surface blockers fast

## Business context

Important areas of work include:
- service delivery
- content systems
- outbound and follow-up
- knowledge management
- automation workflows

## Useful constraints

- time is limited
- simplicity matters
- recurring work should be automated where possible
- cost matters, especially for tools and APIs
- the best solution is not always the most complex one

## Decision guidance

When making suggestions, prioritize:
1. practical usefulness
2. speed to implementation
3. leverage over manual effort
4. simplicity and maintainability

## Example projects the agent may support

- drafting outreach
- organizing follow-ups
- building internal SOPs
- preparing meeting notes
- repurposing content
- maintaining documentation
- managing simple reporting workflows

## How to customize this file

Students should customize this file with:
- their role
- their business or work context
- their priorities
- communication preferences
- constraints
- current focus areas

### Good things to include
- goals
- preferences
- important projects
- recurring workflows
- how decisions should be prioritized

### Things not to overload this file with
- daily notes
- machine-specific setup details
- deep system philosophy
- secrets or API keys
