# MEMORY.md Example

## Purpose

This file stores curated long-term memory.

It should contain the important context the agent should keep across sessions — not every little detail from every day.

A strong `MEMORY.md` makes the agent more useful over time because it preserves the things that actually matter.

---

# MEMORY.md

## User preferences

- Prefers concise, practical answers
- Likes bullet points more than long essays
- Wants clear recommendations, not just open-ended options
- Values speed, leverage, and maintainability

## Working style

- Moves quickly and prefers momentum over perfection
- Wants the agent to reduce friction and save time
- Appreciates proactive help when it is grounded and useful
- Does not want generic advice or unnecessary theory

## Business context

- Runs a service and automation business
- Uses content, systems, and operations as growth levers
- Cares about revenue, delivery quality, and repeatable workflows

## Current strategic priorities

- improve client delivery systems
- increase consistency of follow-up
- make content production more repeatable
- build operational leverage through automation

## Important operating rules

- Recurring work should be systemized when possible
- Simpler solutions are usually better unless complexity clearly creates value
- Important results should be verified, not assumed
- External or destructive actions should usually be confirmed first

## Key tools and systems

- Uses Google Workspace for email, calendar, docs, and drive
- Uses GitHub for repo management and backups
- Uses local and cloud tools for automation workflows

## Important lessons learned

- Wrong-account auth is a common failure point in integrations
- Daily notes belong in dated memory files, not in long-term memory
- Strong startup context improves the usefulness of the agent significantly

## Useful durable context

- Prefers practical systems over abstract frameworks
- Likes progress to be visible and concrete
- Wants reusable workflows, not just one-off wins

## How to customize this file

Students should put in this file:
- durable preferences
- important decisions
- long-term goals
- operating rules worth preserving
- business context that stays relevant
- lessons that should not be forgotten

### Good things to include
- stable preferences
- lasting project or business context
- key decisions
- repeated lessons

### Things not to include
- noisy daily logs
- temporary reminders
- random meeting notes
- machine-specific setup details
- secrets in plain text

### Good rule of thumb
If the information will still matter in a month or two, it may belong here.
If it only matters today, it probably belongs in a daily memory file instead.
